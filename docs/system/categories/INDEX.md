## How to Use

**Classifying a request:**
Every incoming request gets assigned a primary category (0–8) and optionally a secondary. The primary drives Orchestrator routing. The secondary informs which supporting agents join.

**Tagging work:**
All project documents, reports, and outputs carry a category tag. This makes everything findable and sortable by domain.

**Coverage check:**
If a piece of work doesn't fit into one of these nine, the classification is wrong — not the system. Everything fits.
